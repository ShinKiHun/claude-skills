# Handoff 템플릿 (새 폴더 세팅용)

## 1. CLAUDE.md 상단 resume 헤더 블록 (복붙)

```markdown
> **세션 연속성 (handoff 스킬)**: 
> - **시작 시** ("이어서"/"어디까지 했지"): `STATUS.md`(현재상태·다음수) + `LOG.md` 최신 엔트리 1개 + `MARKS.md` 읽고 + `squeue`로 도는 잡 확인 → "여기까지, 다음 X" 요약부터.
> - **끝낼 때** ("저장"/"핸드오프"): `LOG.md`에 오늘 append + `STATUS.md` 덮어쓰기(다음 수 명시).
> - **월간** ("월간 정리"): LOG archive + 오래된 세션 transcript 정리.
> ⚠️ `find`/`du` 트리 전체 스캔 금지(페이지캐시 렉). 무거운 계산은 sbatch(계산노드)만. 로그인노드 heavy 금지.
```

## 2. STATUS.md 템플릿 (1페이지 고정, 매 세션 덮어쓰기)

```markdown
# STATUS — YYYY-MM-DD

## 지금 어디 (한 줄)
<프로젝트가 지금 어느 지점인지 한 문장>

## 진행 중 (도는 잡 + 대기 결과)
| Job | 뭐 | 상태 | 완료 시 판정 |
|---|---|---|---|
| <id> | <설명> | RUNNING/PENDING | <무엇을 보고 판단> |

## 즉시 다음 수 (우선순위)
1. <가장 먼저 할 것 — 구체적으로>
2. <그다음>
3. <선택>

## 열린 질문 / 막다른 길 (반복 방지)
- <아직 못 정한 것 / 시도했다 실패한 것 + 이유>

## 최근 산출물 (파일 경로)
- <path> — <뭐>
```

## 3. LOG.md 엔트리 포맷 (append-only)

```markdown
## YYYY-MM-DD (한 줄 요약, 세션 성격)
- **★ 핵심 결과**: <결정/수치/Job ID>
- <한 것들 ...>
- **막다른 길**: <실패한 것 + 왜> (반복 방지)
- 서버: <GPU 제출 잡 / 로그인노드 heavy 여부>
```

## 4. 파일 역할 요약
| 파일 | 갱신 | 원칙 |
|---|---|---|
| CLAUDE.md | 드묾 | 정적 지식 + resume 헤더 (자동 로드) |
| STATUS.md | 매 세션 | **덮어쓰기** (항상 현재만, 1페이지) |
| LOG.md | 매 세션 | **append** (역사 보존) |
| MARKS.md | 능력 도달 시만 | 실패·반복은 안 올림 |
