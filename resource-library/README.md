# Resource Library

이 폴더는 외부에서 만든 Skill, MCP 서버, 플러그인, 도구와 설치 가이드, 그리고
재사용 가능한 디자인 레퍼런스를 관리한다. 실행 코드나 원문 전체를 복제하는 곳이
아니라, 무엇이 어떤 작업에 유용한지와 적용 전 확인사항을 기록하는 색인이다.

## `skills/`와의 차이

- `skills/`: 우리가 직접 관리하고 설치기로 배포하는 실행 가능한 Skill
- `resource-library/`: 외부 리소스와 우리가 정리한 색상·시각 시스템 같은 재사용 레퍼런스
- `prompt-library/`: 새 프롬프트를 작성할 때 참고하는 프롬프트와 제작 패턴

외부 Skill을 실제 작업에서 반복 검증하고 우리가 유지보수하기로 결정하기 전에는
`skills/`로 복사하지 않는다. 링크, 버전, 라이선스와 설치 명령이 바뀔 수 있기 때문이다.

## 사용법

사용자는 다음처럼 목적만 말하면 된다.

> 외부 리소스 목록에서 프론트엔드 디자인 품질을 높일 Skill과 검수 도구를 찾아줘.

> 논문 리뷰 발표에 어울리는 색상 조합을 골라줘. 본문 텍스트가 읽히는 조합만 써줘.

에이전트는 [catalog.yaml](catalog.yaml)에서 관련 항목을 찾고, 설치가 필요하면 원본
저장소와 현재 공식 문서를 다시 확인한다. 토큰이나 API 키가 필요한 도구는 비밀값을
저장소나 명령 기록에 남기지 않는다.
