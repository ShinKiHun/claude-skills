# Prompt Library

이 폴더는 자동 실행되는 Skill이 아니라, 프롬프트와 제작 패턴을 축적하고 평가하는
자료실이다. 에이전트는 현재 작업과 관련된 파일만 선택해서 읽어야 한다.

사용자는 파일명이나 번호를 기억할 필요가 없다. 예를 들어 “연구 발표자료용 프롬프트를
작성할 건데 참고할 만한 프롬프트를 골라줘”라고 요청하면, 에이전트가
`catalog.yaml`의 `reference_for`와 `tags`를 기준으로 후보를 찾는다.

## 분류

- `inbox/`: 아직 사용하거나 평가하지 않은 자료
- `catalysis/`: 계산촉매, 표면과학, DFT 및 계산과학에 직접 관련된 자료
- `cross-domain/`: 분야는 다르지만 구조, 품질 기준, 제작 방식이 유용한 자료
- `external/`: 공개된 다른 사람의 프롬프트 또는 링크 기반 자료
- `archive/`: 중복, 구버전, 저품질 또는 더 이상 사용하지 않는 자료

사용 빈도는 폴더로 나누지 않고 각 파일의 `frequency` 메타데이터로 관리한다.
분야와 빈도는 서로 다른 기준이기 때문이다.

외부 글의 핵심이 프롬프트 본문이 아니라 Skill, MCP 또는 도구 설치법이라면 이곳이
아니라 `../resource-library/`에 넣는다. 예시 프롬프트가 몇 줄 포함되어 있어도 자료의
주된 용도로 분류한다.

## 참고 자료 선택 규칙

1. 현재 요청의 산출물과 작업 유형을 확인한다.
2. `catalog.yaml`에서 `workflow`, `reference_for`, `tags`가 가까운 항목을 찾는다.
3. 가장 관련 있는 자료를 최대 세 개만 읽는다.
4. 원문을 합치거나 표현을 무작정 복사하지 않고 재사용 가능한 구조만 추출한다.
5. 과학적 사실과 실제 입력은 참고 프롬프트가 아니라 사용자 자료를 기준으로 한다.
6. 답변에 사용한 prompt ID와 가져온 핵심 패턴을 짧게 밝힌다.

## 자료 수명 주기

1. 발견한 자료를 `inbox/`에 저장한다.
2. 출처, 작성자, 수집일과 사용 조건을 기록한다.
3. 실제 작업에서 시험하고 결과를 남긴다.
4. 효과가 있던 문장보다 효과가 난 이유를 추출한다.
5. 범용 원칙은 `cross-domain/`, 분야 지식은 `catalysis/`로 이동한다.
6. 반복적으로 효과가 확인된 절차만 `../skills/`로 승격한다.
7. 중복되거나 뒤처진 자료는 삭제 대신 먼저 `archive/`로 이동한다.

## 외부 프롬프트

공개된 프롬프트를 저장할 때 원문을 자신의 작업처럼 표시하지 않는다. 최소한 다음을
기록한다.

- 원문 URL과 작성자
- 수집일
- 라이선스 또는 재사용 조건을 확인했는지 여부
- 전체 원문인지 요약·발췌인지
- 어떤 구조가 유용한지
- 어떤 상황에서는 사용하면 안 되는지

권리가 불분명하면 원문 전체보다 링크와 구조적 요약을 보관한다.

새 파일은 [_template.md](_template.md)를 복사해서 작성한다. 전체 색인은
[catalog.yaml](catalog.yaml)에 기록한다.
