# meeting-report — 미팅 보고자료 생성 스킬

연구 프로젝트의 주간 미팅(예: 매주 수요일 교수님 1:1)용 보고자료를 만든다.
두 가지 모드:

- **"자료 제작"** (당일): 오늘 세션에서 한 작업 중 중요한 것만 추려 하루치 HTML
- **"총자료 제작"** (미팅 전날): 그 주의 일일 리포트들을 종합해 특히 중요했던
  내용만 선별한 주간 HTML

## 1. 폴더/파일 규칙

모든 리포트는 **Claude Code를 실행한 현재 디렉토리** 기준 `reports/` 아래:

```
reports/
└── 20260701_20260707/     ← 주간 폴더 (시작일_종료일, 7일 창)
    ├── 20260701.html      ← 일일 리포트 (당일 날짜)
    ├── 20260703.html
    └── total.html         ← 주간 종합 (총자료)
```

**주간 폴더 결정 알고리즘** (자료 제작 시 매번 수행):
1. `reports/` 안의 `YYYYMMDD_YYYYMMDD` 폴더 중 가장 최신을 찾는다.
2. 오늘이 그 폴더의 날짜 범위 안이고 `total.html`이 **없으면** → 그 폴더 사용.
3. 아니면 (범위 지남 / total 이미 생성 / 폴더 없음) → 새 폴더
   `<오늘>_<오늘+6일>` 생성. (직전 total 다음날부터 새 주가 시작되는 효과)
4. 총자료 제작은 항상 현재 활성 주간 폴더에 `total.html`로. 이미 있으면
   덮어쓰기 전에 사용자에게 확인.

## 2. 내용 수집

### 일일 ("자료 제작")
우선순위 순서로 수집:
1. **현재 세션 대화** — 오늘 실제로 한 일, 나온 수치, 내린 결정 (1차 소스)
2. 프로젝트 일지 파일 (`LOG.md`, `*.log.md`, `journal*` 등이 있으면 오늘 날짜 항목)
3. `git log --since=midnight --oneline` (git repo인 경우)
4. 오늘 수정된 결과 파일 (`find . -newermt today -name "*.json" -o -name "*.png"` 등,
   과하게 뒤지지 말고 세션에서 언급된 것 위주)

**선별 기준** — 전부 넣지 않는다. 다음에 해당하는 것만:
- 결론이 바뀌거나 새로 선 것 (신기록, 판정, 가설 확정/기각)
- 미팅에서 질문받을 수치 (표로)
- 실패/막힌 것도 정직하게 (원인 진단이 있으면 함께)
- 다음 단계

### 주간 ("총자료 제작")
1. **활성 주간 폴더의 일일 HTML들이 1차 소스** — 다른 세션에서 실행해도
   일주일치가 복원되도록. 각 일일 HTML을 읽고 핵심을 추출.
2. 현재 세션 컨텍스트로 보강 (당일 작업이 아직 일일 리포트에 없으면 포함).
3. **"특히 중요했던 것"만** — 미팅에서 5–10분에 설명 가능한 분량으로 압축.
   일일 리포트의 나열이 아니라 주 단위 스토리로 재구성:
   무엇이 진전됐나 → 근거 수치 → 남은 리스크 → 다음 주 계획 → 논의 필요 사항.

## 3. HTML 규격 (self-contained 필수)

- **단일 파일**: 인라인 CSS, 외부 리소스 0 (CDN/웹폰트 금지) — 오프라인/이메일
  첨부에서도 열려야 함
- **이미지**: 관련 그림 파일은 base64 data-URI로 임베드. 큰 그림은 임베드 전
  리사이즈 (PIL 등, 이미지당 ~500KB 이하 목표). 이미지 없으면 표/수치로 충분
- **구조** (일일):
  1. 헤더 — 프로젝트명 / 날짜 / "Daily Report"
  2. **한 줄 결론** (오늘의 TL;DR — 큰 글씨)
  3. 핵심 결과 — show-me 스타일: 실제 수치가 든 표 + 그림. 항목당 소제목 + 2-3문장
  4. 이슈/실패 (있으면)
  5. 다음 단계
- **구조** (주간 total): 위에 더해
  6. **미팅 논의 포인트** — 지도교수 결정이 필요한 항목 명시
- **스타일**: 시스템 폰트 스택, 최대폭 ~880px 중앙정렬, 라이트 배경, 표는
  hairline 테두리 + 우측정렬 숫자, 인쇄 친화 (`@media print` 고려),
  기술용어는 원어 유지. 언어는 사용자가 쓰는 언어(기본 한국어)
- 참고 마크업: 이 스킬 폴더의 `template.html` (구조와 CSS를 그대로 재사용 권장)

## 4. 작성 태도

- 본론만. 미사여구·과장 금지. "~했음/~로 판정" 톤
- 모든 주장 옆에 근거 수치 (show-me)
- 부정적 결과도 보고 — 원인 진단이 붙으면 그게 성과
- 리포트 생성 후 사용자에게 파일 경로와 한 줄 요약만 알려주면 됨
